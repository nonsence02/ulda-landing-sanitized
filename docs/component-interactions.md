# Component Interactions

## Загальна схема

Лендинг побудований навколо одного головного маршруту `app/page.tsx`, який оркеструє контент, локалізацію та складені UI-блоки.

## Основні взаємодії

### `app/layout.tsx`

- підключає глобальні стилі;
- надає metadata і viewport;
- обгортає застосунок у `ThemeProvider`;
- додає skip-link для доступності.

### `app/page.tsx`

- читає контент із `lib/thesis-content.ts`;
- використовує `LanguageSwitcher` для перемикання між `uk` і `en`;
- передає локалізовані пропси в `Footer`;
- збирає сторінку з карток, кнопок та допоміжних UI-компонентів.

### `components/language-switcher.tsx`

- отримує поточну мову й callback;
- не зберігає власний контентний стан;
- впливає лише на вибір поточного локалізованого набору даних.

### `components/footer.tsx`

- залежить від `siteConfig` для посилання на репозиторій;
- отримує локалізовані підписи з `app/page.tsx`.

### `components/ui/*`

- `button.tsx`, `card.tsx`, `tooltip.tsx` надають базові примітиви для узгодженого інтерфейсу;
- не містять knowledge про тему бакалаврської роботи;
- повторно використовуються різними секціями сторінки.

### `lib/site.ts`

- слугує джерелом metadata і deployment URL;
- використовується в `app/layout.tsx`, `app/robots.ts`, `app/sitemap.ts` та `components/footer.tsx`.

### `lib/thesis-content.ts`

- містить повний структурований двомовний контент;
- використовується в `app/page.tsx` як єдине джерело академічного текстового наповнення.
