# Генерація документації

## Призначення

У репозиторії використовується `TypeDoc` для побудови HTML-документації на основі TSDoc-коментарів у TypeScript/TSX-модулях.

## Основна команда

```bash
npm run docs:build
```

Команда читає конфігурацію з `typedoc.json` і генерує статичний сайт документації в папку `typedoc-site/`.

## Швидка перевірка

```bash
npm run docs:check
```

Зараз ця команда виконує ту саму генерацію й зручна як окремий quality gate для CI або локальної перевірки перед комітом чи PR.

## Що включено в автоматичну документацію

- `app/layout.tsx`
- `app/page.tsx`
- `app/robots.ts`
- `app/sitemap.ts`
- публічні експорти з `components/`
- публічні експорти з `lib/`

## Що виключено

- build-артефакти;
- CSS-файли;
- тимчасові директорії на кшталт `.next/`, `out/`, `typedoc-site/`.

## Практика підтримки

- додавай TSDoc до всіх нових публічних експортів;
- якщо змінюється поведінка або роль модуля, оновлюй коментарі разом із кодом;
- при зміні архітектури або потоків даних синхронізуй `docs/architecture.md` та `docs/component-interactions.md`.
